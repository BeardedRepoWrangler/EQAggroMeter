# 2026-05-04

## What changed

- **Adopted semantic versioning. v1.0.0 released.**
- New module `lua/aggrometer/version.lua` — single source of truth for the
  version. Exports `MAJOR`, `MINOR`, `PATCH`, `PRERELEASE`, `string()`,
  `display()`.
- `ui.lua` footer now renders `mode: …   members: …   v1.0.0` in dim color,
  always (even when roster is empty / no target). Style chosen to mirror the
  EQXPInfo project standard (lowercase v, three-part semver, dim gray,
  inline with the status line).
- `init.lua` — added `/agm version` (alias `/agm ver`) subcommand and folded
  the version into the script-load chat banner.
- New ADR — [[decisions/0008-semantic-versioning|ADR 0008]] documents the
  bump policy (MAJOR / MINOR / PATCH classifications specific to this
  project), the surface (footer, slash command, banner), and the rationale
  for the chosen approach.
- New runbook — [[runbooks/cut-a-release]] — step-by-step release ritual:
  bump `version.lua`, update `CHANGELOG.md`, commit, tag, push tag.
- New `CHANGELOG.md` in Keep-a-Changelog format. Seeded with a v1.0.0 entry
  summarizing pre-1.0.0 work; older detail lives in `decisions/` and
  `log/`.
- Updated `CLAUDE.md` — added operating rule #10 (versioning is a single
  source of truth, see ADR 0008 + cut-a-release runbook), extended
  hygiene rule #9 to require a version-bump proposal at every meaningful
  change.
- Updated `Index.md`, `README.md`, `Roadmap.md` — link the new docs and
  surface the v1.0.0 release.

## Why

The project is being shared across characters and bug reports came back
without a way to identify which build was running. Semver also gives us a
clear signal for wire-protocol breakage risk between peers: MAJOR =
incompatible, MINOR = additive (peers ignore unknown lines safely), PATCH
= no observable change.

The footer placement (vs. window title) was chosen to avoid breaking
ImGui.ini state on every bump (the title is the window key) and to match
the EQXPInfo styling Michael flagged as the standard.
