# Design

Living design docs — the current best understanding of how each piece of the system works or should work. Unlike ADRs (which are immutable once accepted), design docs are updated in place as the design evolves.

## What belongs here

- Component-level designs (parser, threat model, UI)
- Data flow and architecture overviews
- Open questions and trade-offs that aren't yet ADR-worthy

Use [[templates/design-doc]] when starting a new one.
